# Amendments G20–G23 — text I/O, alphabet learning, free-form text babble

**Status: pre-registered. Not implemented. Gated.**
**Frozen: 2026-05-11. Author: Michael Kupermann.**

This document is the full, pre-registered specification for the English-text
chain of amendments. It exists *before* any G20 code is written, so the
acceptance bars cannot be retro-fitted to the runs that test them. The
project's existing discipline (`docs/marker_protocol.md`) governs every line
of this document.

If a future reader finds this file edited after any of G20–G23 has been run
against it, that is a protocol violation and the run's verdict is void.

---

## 0. Why this exists

The substrate currently has audio and video ports, an STDP/BTSP plasticity
chain, dream consolidation, cross-modal recall (G13), and a predictive-babble
production loop (G19). It has no text port and cannot write back. This
chain adds text — staying strictly inside the project's stated mechanisms:
no LLM, no learned tokeniser, no pretrained embedding, no transformer, no
backprop. The text port is engineered topology (`docs/CONCEPT.md` §4.8), the
same kind of engineered axonal projection that audio and video already use.
The atoms inside the text port and the bridges leaving it must form via the
substrate's own binding + STDP + BTSP + dream rules. If they don't, the
amendment FAILS.

---

## 1. The gate (Q3 = C, locked)

G20 does not start until **all three** of the following hold simultaneously:

1. The G19 predictive-babble verdict is **PASS** on a real-corpus 24-hour
   run, under the pre-registered ≥2 σ bar in `docs/predictive-babble.md`.
2. The G16/G17 five-marker conjunction fires under the trained-engram run,
   per `docs/marker_protocol.md`.
3. The matched-config negative control does **not** fire the markers
   (`max_markers_seen < 5`).

This gate is **indefinite**. There is no fallback. If G19 never PASSes, G20
never starts. Building text I/O on a substrate that cannot yet pass its
existing audio-side acceptance is engineering theatre; it would prove
nothing about whether the substrate can hold symbols, only that we can move
firings into and out of a sub-cube.

Explicitly accepted by the author on 2026-05-11.

## 2. Frozen design decisions

These are not up for discussion once G20 begins. If a decision is
later overturned, the amendment chain is renumbered (G20 → G20′) and the
old chain remains in the record as written.

### 2.1 End state (Q1 = D)

Three concrete results define success across the chain:

- **G21 = smoke.** Substrate round-trips a single letter (see→hear→write).
- **G22 = demo.** Substrate holds a 5-letter set with bounded interference.
- **G23 = scientific.** Substrate produces open-loop text babble that
  statistically resembles its training corpus and not white noise.

### 2.2 Training sources (Q2)

Letters are trained tri-modally: video (the letter shape), audio (the
spoken letter), text (the symbol on the text port). Sources, in order of
use:

- **Bulk (B + C):** synthetic-rendered letter images (PIL fonts) + macOS
  `say` command (or any non-neural TTS) for audio + the typed symbol;
  EMNIST + a public spoken-alphabet corpus for variation.
- **Personalisation (A), stage-4 analogue:** webcam + microphone recording
  of the author producing each letter. Same role as G19 stage 4.
- **English first.** German is a separate amendment chain (G24–G26, see §7).

### 2.3 Reward protocol (Q4 = D)

During bulk training (B + C): reward is computed automatically from
dataset ground-truth. Match → `+1` via the existing Plan E `s_reward_polarity` /
`k_reward_polarity` tristate. Mismatch → `−1`. No human in the loop.

During stage-4 personalisation: a single human keypress per trial in the
interactive viewer maps to the same tristate. Space = correct (+1), x = wrong
(−1), no input = neutral (0). The credit-assignment window is fixed at 200 ms
sim time (12 ticks at `dt = 1/60`). No graded reward.

### 2.4 Port topology (Q5 = A)

The text port occupies **27 disjoint 6×6×6 sub-cubes** arranged in a 3×3×3
macro-grid in one corner of the 60×60×60 substrate. One sub-cube per
symbol (a–z + space).

```
sub-cube origin =
    (x0 + 6 * (idx %  3),
     y0 + 6 * (idx // 3) % 3,
     z0 + 6 * (idx // 9))     # idx ∈ [0, 26]
text port corner: (x0, y0, z0) = (0, 0, 0)
total volume occupied: 18×18×18 (a 1/37 fraction of the substrate)
```

The decoder is:

```python
def read_symbol(world, window_ticks=6):
    # argmax over sub-cubes of "alive atoms whose pos is inside this sub-cube
    # AND which fired within the last window_ticks ticks"
```

No readout layer. No learned classifier. No parameters except `window_ticks`,
which is frozen at 6 (= 100 ms at `dt = 1/60`).

**Cross-symbol bridges that form without co-stimulation are measurements,
not bugs.** With `r_2 = 10` under the session-3 calibration and sub-cube
edge 6, two adjacent sub-cubes' atoms can be within binding range across
the shared face. If sub-cube `A` consistently co-excites sub-cube `B`, that
is the substrate reporting which symbols its own geometric rules consider
similar. We do not relax `r_2`, redesign the topology, or filter the cross-
symbol bridges out. The phenomenon is logged and treated as data.

### 2.5 Forgetting strategy (Q7 = E)

Primary hypothesis for G22: **C or D** — single substrate, no `pattern_id`
segregation; the dream consolidation phase (G15/G18) and/or BTSP eligibility
traces (G14) carry forgetting-resistance. This is the project's actual
scientific claim about minimal-substrate memory.

Fallback: **B** — single substrate with `k_pattern_id` segregation per
letter (one pattern_id per symbol). If C/D fails at ceiling and B carries,
the G22 verdict is **PASS (qualified)** and labelled as such permanently in
the README status table, this document, and any external write-up.

Excluded: **A** (SubstrateLibrary, 27 parallel substrates) is not a
G22 path. It remains a production tool for non-research applications.

### 2.6 Failure honesty (Q9 = C, with retry rule)

FAILED amendments are first-class. The README status table shows FAILED
rows next to PASSed rows. The interactive GUI surfaces the failure state.
A post-mortem markdown is written for every FAILED amendment in
`docs/amendments/G{N}-postmortem.md`.

**Retry rule.** A FAILED amendment's verdict is permanent. Its acceptance
bar is never edited. The author may design a new amendment with a new
number (G21′, G21″, …) with its own pre-registered acceptance and try
again, indefinitely. This is "we don't stop until it works" reconciled with
the project's no-moving-targets discipline. Each retry is a separate,
named, fail-able event.

### 2.7 Budget (Q6 = D)

Hybrid budgets. Wall-clock on the author's Mac (macOS-arm64, Python 3.13).
Clock starts when the amendment's first commit lands; clock pauses if the
gate (§1) is not yet met.

| Amendment | Realistic | Hard ceiling — FAILED if exceeded |
|---|---|---|
| G20 — text port primitive | 2 weeks | 4 weeks |
| G21 — single-letter `A` round-trip | 4 weeks | 8 weeks |
| G22 — 5-letter set + forgetting | 12 weeks | 24 weeks |
| G23 — open-loop text babble PASSes 2σ | 16 weeks | 32 weeks |

At ceiling: post-mortem written within 7 days, verdict FAILED, retry under
a new amendment number per §2.6.

## 3. G20 — text port primitive

**Scope:** plumbing only. No learning.

### 3.1 What is built

- `agent/text_io.py`:
  - `TextIO(world, x0=0, y0=0, z0=0)` constructor places the 27 sub-cubes.
  - `write_symbol(c: str) -> None` injects vibrations into the sub-cube for
    character `c`. Number, frequency distribution, and lifetime of injected
    vibrations match the `audio_io` injection contract (same shape; same
    per-tick budget).
  - `read_symbol(window_ticks: int = 6) -> Optional[str]` returns the
    symbol whose sub-cube held the most atoms that fired within
    `window_ticks` of `world.t`. None if no firings.
  - `reset()` clears all 27 sub-cube interiors.
- A `text_input` and `text_output` register in the world's port table,
  parallel to `audio_input` / `audio_output`. Symbol injection writes to
  `text_input`'s region; symbol readout reads from `text_output`'s region.
  These may be the same region in G20 (single port) and split in a later
  amendment if asymmetric topology is required.

### 3.2 Acceptance — pre-registered

Test file: `tests/test_amendment_G20_text_port.py`. All four must pass:

1. **Geometric correctness.** All 27 sub-cubes are disjoint, each is
   exactly 6×6×6, and the union fits inside the substrate. Asserted by
   construction-time geometry check.
2. **Write-only smoke.** Calling `write_symbol('a')` 30 times over 1 sim-
   second results in `n_alive` vibrations within the `a` sub-cube being
   nonzero and within the `b` sub-cube being zero. (No binding, no learning
   yet — pure injection.)
3. **Read-only smoke.** Manually allocating an atom inside the `a` sub-
   cube and forcing it to fire produces `read_symbol() == 'a'`. Doing
   the same with two atoms (one in `a`, one in `b`, the `b` atom firing
   more) produces `read_symbol() == 'b'`.
4. **Round-trip without binding.** `write_symbol('a')` for 1 sim-second
   followed by `read_symbol()` returns `None` *or* `'a'` — never any other
   letter. ("None" is acceptable here because G20 explicitly does not
   train; whether injected vibrations bind into an atom in 1 sim-second
   depends on the calibration, and that is G21's question, not G20's.)

No bridges, no STDP, no BTSP, no dream. G20 PASSes only if it's purely
mechanical I/O.

### 3.3 What G20 does **not** prove

It does not prove that the substrate can learn letters. It proves only
that the topology and the decoder one-liner do what the spec says they do.

## 4. G21 — single-letter `A` round-trip

**Scope:** the smoke test for the whole claim.

### 4.1 Curriculum

20-minute sim-time training on a single letter, `a`:

- Every 200 ms sim time, a tri-modal training pulse fires:
  - video port receives the EMNIST glyph for `a` (one of ~5000 variants,
    sampled with seed),
  - audio port receives a 200 ms clip of the spoken English `a` (from a
    public spoken-alphabet corpus, randomly drawn),
  - text port receives `write_symbol('a')`.
- Reward channel fires `+1` uniformly on all bridges that fired in the
  preceding 200 ms (the credit-assignment window from §2.3).

After the 20-minute training, a 1-minute test phase:

- 20 trials. In each trial, one of the three modalities is presented and
  the other two are read. The substrate must produce the matching output
  on the two un-stimulated modalities.

### 4.2 Acceptance — pre-registered

- **PASS:** at least 16 of 20 trials succeed (i.e. ≥80%). A trial succeeds
  when *both* un-stimulated modalities report `a` within the test window.
- **Negative control:** a matched-config substrate trained on `b` instead
  of `a`, tested on `a`-stimuli, must succeed on ≤ 2 of 20 trials
  (chance for 27 symbols ≈ 3.7 %; binomial 2 σ on 20 trials ≈ 12 %).
- **Discriminating test:** PASS rate − control rate must be ≥ 68 percentage
  points. (80 − 12.)

A run that PASSes the trained bar but the control also PASSes, or fails to
beat the control by the 68 pp margin, is **NULL**, not PASS. NULL is a
real outcome — see `docs/predictive-babble.md` for the precedent.

### 4.3 Failure modes pre-registered as expected

- **No atom forms in the `a` sub-cube within the training window.** Likely
  cause: vibration density inside a 6×6×6 sub-cube is too low for the
  binding rule. Honest response: increase the per-pulse vibration count,
  not the sub-cube size or the binding radius. Geometry is frozen.
- **Atoms form in the `a` sub-cube but no cross-modal bridges form.** Likely
  cause: 18×18×18 corner is too far from audio_input / video ports.
  Honest response: G21 FAILS, post-mortem, new amendment (G21′) re-locates
  the text port in physical space, with new pre-registered acceptance.
- **Cross-modal bridges form but don't survive the test window.** Likely
  cause: STDP/BTSP parameters aren't right for the new timescales. Honest
  response: G21 FAILS, post-mortem, retry under G21′ with re-tuned
  plasticity (which itself becomes pre-registered before the G21′ run).

In all three cases: do not edit G21's acceptance bar. Renumber.

## 5. G22 — 5-letter set + forgetting protocol

**Scope:** the demonstration.

### 5.1 Curriculum

Train `a`, `b`, `c`, `d`, `e` sequentially. Each letter gets 20 minutes
sim time of training, identical shape to G21. After all five are trained,
20-trial round-trip tests on each letter, in random order.

### 5.2 Acceptance — pre-registered

- **PASS (unqualified)** if, using strategy C or D from §2.5 (no
  `pattern_id` segregation):
  - Each of the 5 letters round-trips at ≥80% over its 20 trials, AND
  - The earliest-trained letter (`a`) shows ≤10% drop relative to its
    G21-style baseline measured immediately after training.
- **PASS (qualified)** if C and D both fail at ceiling, and strategy B
  (`k_pattern_id` per-letter segregation) makes the above thresholds hold.
- **FAILED** if neither path works at ceiling.
- **Negative control:** same shape — matched-config substrate trained on
  letters `f`–`j`, tested on `a`–`e`, must round-trip ≤ chance + 2σ.

### 5.3 What gets reported

Whatever the outcome: this document, the README status table, the
post-mortem (if FAILED), and the medium write-up (if any) all carry the
same verdict and the same qualification. No selective reporting.

## 6. G23 — open-loop text babble

**Scope:** the long-horizon scientific result. Direct analogue of G19.

### 6.1 Production protocol

After G22 PASSes (qualified or not), the substrate is trained on a real
English text corpus (the same corpus used for G19's audio side, decoded to
its character sequence — Project Gutenberg + audiobook transcripts; spec
locked in `docs/predictive-babble.md`). Training duration matches G19's
4-stage curriculum.

After convergence: input ports are gated off. The substrate runs free for
5 sim-minutes. `read_symbol()` is polled every 100 ms. The resulting
character string is the substrate's babble output.

### 6.2 Acceptance — pre-registered

Four substrates: trained-EN, white-noise, reversed-EN, French.
Matched-wall-clock per stage, per G19's protocol. Evaluator computes
character-trigram histogram KL-divergence against held-back EN text.

- **PASS:** trained-EN beats every control by ≥ 2 σ on bootstrap (100
  bootstraps), AND `trained_kl_mean < every control_kl_mean`.
- **FAIL:** z-score against white-noise ≤ 0.
- **NULL:** anything else; `null_against` lists which controls were
  inconclusive.

Output JSON shape: identical to G19's `result.json` (see
`docs/predictive-babble.md` §"Reading result.json"), substituting
`character-trigram-KL` for `MFCC-KL`.

### 6.3 Compute budget

Pre-registered: the run completes on the author's Mac within 7 days
wall-clock or G23 FAILs. (G19 budget was 24 hours; text babble is
expected to be slower per sim-second because three ports run instead of
two, but produces less data per sim-second. 7 days is the worst-case
realistic ceiling, not an invitation.)

## 7. German chain (G24–G26) — pre-registered as future

**Numbering:** parallel chain. G24/G25/G26, not G20-de/G21-de.

- **G24** — German on **fresh substrate**. Acceptance: G23-shape PASS on
  German corpus, against German white-noise / reversed-DE / French
  controls. Does not require English knowledge.
- **G25** — German on **fork of English-PASSed substrate**. Acceptance:
  G23-shape PASS on German, AND convergence wall-clock < (G24's wall-
  clock × 0.7). Demonstrates transfer learning, not continual learning.
- **G26** — **Continual learning.** Train German on the English-PASSed
  substrate without forking. Acceptance: G24's PASS on German AND
  English round-trip degrades ≤10% relative to pre-German baseline.

G26 is **expected to FAIL on the first attempt.** This is recorded here so
that a future FAIL is not surprise news, and a future PASS is not
overclaimed. If G26 FAILs at ceiling, G26′ is designed with whatever
plasticity-protection mechanism the post-mortem motivates. Per §2.6 the
G26 verdict remains FAILED; G26′ is a new, separately-pre-registered
amendment.

## 8. Toolset enhancement (Q10b = (ii), locked)

In parallel with G20–G23, the project ships an **amendment-scaffolding CLI**:

```
eqmod new-amendment G21
    → docs/amendments/G21.md                  (spec stub, pre-registered)
    → docs/amendments/G21-postmortem.md       (template, used if FAILED)
    → tests/test_amendment_G21_*.py           (test skeleton)
    → docs/marker_protocol.md addendum block  (pre-registered thresholds)
    → README.md status table row              (status = PRE-REGISTERED)
```

The CLI enforces the discipline by making the discipline the path of least
resistance. It does **not** auto-write the implementation. It writes only
the pre-registration scaffolding. The implementation is still authored
manually, against the frozen spec.

This CLI is a deliverable in parallel to G20. It is not gated on G20–G23.

## 9. Reusable-mechanisms register (Q9 amendment, locked)

When the substrate work surfaces a generalisable mechanism — a way for a
system to learn, a deadlock-breaking pattern, a topology trick — it is
written into `docs/patterns/` as a standalone markdown, *not* buried in
the substrate code. Each entry includes:

1. The mechanism in technical detail.
2. The empirical evidence that produced it.
3. The minimal reusable form (no substrate dependencies).
4. A business / real-world mapping — where this mechanism could be
   applied outside the substrate.

This is the explicit operationalisation of the README's claim that the
"patterns I am building to attack it are already shippable." Patterns are
public, not hidden in `world/physics.py`. See `docs/patterns/README.md`
for the index.

## 10. Out of scope

Listed here so that future scope-creep is visible against this document.

- No LLM, no transformer, no learned tokeniser, no pretrained embedding,
  no backprop in any G20–G23 path.
- No new neural mechanism that is not already in the substrate today
  (STDP, BTSP, dream consolidation, G6 routing, Plan E reward).
- No claim about phenomenal consciousness (Chalmers's hard problem
  remains untouched).
- No claim about understanding, semantics, or grounding. The substrate
  will, at best, demonstrate that engineered binding rules under engineered
  topology produce statistical structure that mirrors training data. That
  is the entire claim. It is small, and it is honest.
